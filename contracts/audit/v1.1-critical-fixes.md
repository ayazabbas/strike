## H-01 Fix: Cross-Contract Reentrancy DoS via ERC1155 Callbacks

**Problem:** During batch settlement, ERC1155 `mintSingle` and `transferEscrowTokens` trigger `onERC1155Received` callbacks. A malicious contract recipient can reenter `OrderBook.cancelOrder()` (whose reentrancy guard is not locked), cancelling a later order in the same batch. When settlement reaches that order, `reduceOrderLots` reverts (lots = 0), reverting the entire batch.

**Alternative Fix — Deferred Token Delivery:**

Instead of minting/transferring ERC1155 tokens inline during settlement, accumulate pending deliveries and process them after all orders are settled. This eliminates callbacks entirely during the settlement loop:

```solidity
// In BatchAuction, accumulate pending mints:
struct PendingMint {
    address to;
    uint256 marketId;
    uint256 lots;
    bool isYes;
}
PendingMint[] internal pendingMints;

// After _settleChunk loop completes:
for (uint256 i = 0; i < pendingMints.length; i++) {
    outcomeToken.mintSingle(
        pendingMints[i].to,
        pendingMints[i].marketId,
        pendingMints[i].lots,
        pendingMints[i].isYes
    );
}
delete pendingMints;
```

## M-03 Fix: Cap GTC Rollover Batch Size

The simplest fix is to cap `pushBatchOrderId` so batches never exceed `SETTLE_CHUNK_SIZE`, eliminating the multi-chunk re-computation issue entirely:

```diff
--- a/src/OrderBook.sol
+++ b/src/OrderBook.sol
@@ -494,8 +494,12 @@ contract OrderBook is AccessControl, ReentrancyGuard, ERC1155Holder {
     /// @notice Push an order ID to a batch's order list (for GTC rollover).
-    ///         No cap — chunked clearBatch handles arbitrarily large batches.
-    function pushBatchOrderId(uint256 marketId, uint256 batchId, uint256 orderId) external onlyRole(OPERATOR_ROLE) returns (bool) {
-        batchOrderIds[marketId][batchId].push(orderId);
+    ///         Returns false if batch is full (caller should auto-cancel the order).
+    function pushBatchOrderId(uint256 marketId, uint256 batchId, uint256 orderId)
+        external onlyRole(OPERATOR_ROLE) returns (bool)
+    {
+        if (batchOrderIds[marketId][batchId].length >= MAX_ORDERS_PER_BATCH) {
+            return false; // Batch full — caller should cancel or skip
+        }
+        batchOrderIds[marketId][batchId].push(orderId);
         return true;
     }
```
